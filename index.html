How I approach frontend development

Every project starts with clarity. I define the outcome, map a single primary user path, then design the UI and architecture around that path.

  • Design-led architecture. Components and routes follow a clear inventory rather than improvisation.
  • Performance from the start. Minimal dependencies, simple layouts, fast first paint.
  • Accessibility baked in. Semantics, focus, labels, and error messaging are part of the initial build.

Stacks I am confident with

On the web, I work comfortably with React, Next.js, Vue, Twig templates, and modern vanilla JavaScript. On mobile, I work with React Native and Expo, backed by Convex or Supabase for real-time data.
